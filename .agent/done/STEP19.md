# STEP19

## 目的

イベント開始時・終了時のリマインダー文面を固定文字列から LLM 生成へ切り替え、当日の会話履歴とユーザー ID を文脈として渡せるようにする。

## 方針

1. `eventReminders` で LINE の送信対象 userId を明示的に扱う。
2. 既存の OpenAI クライアントを使い、固定のリマインダー文面をベースにした短い返答を LLM に生成させる。
3. 当日の会話履歴を読み込み、必要なら一言だけ自然な補足を加えるプロンプトにする。
4. LLM 失敗時は従来の固定文面へ安全にフォールバックする。
5. 生成文のベースメッセージ・フォールバックをテストで担保する。
