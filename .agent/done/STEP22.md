# STEP22

## 目的

`22:00` の夜サマリー送信を廃止し、日次処理を `22:00` 催促、`03:00` 内部締め、`08:00` 朝メッセージへ再編する。  
1日の定義を `03:00` 区切りに統一し、内部サマリーは `log.md` に保存するが LINE 会話履歴へは保存しない。  
X 投稿は `03:00` 締め直後に前日分として実行する。

## 実装内容

1. `night` ジョブを「今日のまとめ送信依頼」専用に変更し、LINE 送信のみ行うようにした。
2. `close` ジョブを新設し、`前日03:00-当日02:59:59` の会話と event をもとに内部サマリーを生成して `log.md` に保存するようにした。
3. `close` ジョブのサマリー本文は LINE に送らず、会話履歴にも保存しないようにした。
4. 朝メッセージ生成は前日会話ではなく、`close` ジョブで確定した前日サマリーと当日予定を主文脈に使うようにした。
5. X 投稿は `close` ジョブ直後に前日分として実行し、notification state の `close` レコードで重複防止するようにした。
6. `README.md`、`.agent/README.md`、API エンドポイント定義、単体テストを新しい3ジョブ構成へ更新した。

## 完了条件

1. `22:00` ジョブは催促 LINE のみを送る。
2. `03:00` ジョブが `log.md` を更新し、内部サマリーを会話履歴へ保存しない。
3. `22:00-03:00` の自由会話が前日分として `03:00` 締めに含まれる。
4. 朝メッセージが前日サマリーと当日予定を使って生成される。
5. X 投稿が `03:00` ジョブで1回だけ実行される。
6. `npm test` と `npm run lint` が通る。
