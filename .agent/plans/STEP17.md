# Step 17: 朝の挨拶文面をLLMが決定する仕様追加

## 目的
朝ジョブ（`morning`）で送るメッセージを固定文面ではなくLLM生成に切り替える前提を定義する。
LLM は前日までの会話履歴、`SOUL.md`、`USER.md`、当日予定を読み、
その日のユーザー向け朝挨拶、予定確認、前向きになれる一言をまとめて返す。

## 固定方針
- Step 17 は仕様定義が主目的であり、機能実装は行わない
- 朝ジョブの文面決定は専用LLMが担当する
- 朝メッセージは少なくとも「挨拶」「今日の予定確認」「元気づける一言」を含む
- 生成文面は `SOUL.md` の人格・口調方針に従う
- `USER.md` の嗜好・状況を踏まえて言い回しを調整する
- 前日会話履歴を読み、その直近文脈に沿った自然な一言を優先する
- 当日予定がない場合でも、予定なし前提の自然な朝挨拶を返せるようにする

## 対象LLM
- MorningGreetingComposer（新規: 朝挨拶文面の生成担当）

## MorningGreetingComposer 仕様
1. 呼び出しタイミング
- 朝ジョブ（`morning`）実行時に毎回呼ぶ

2. 役割
- 朝の第一声となる自然な挨拶を生成する
- 当日予定を短く確認できる形に整理する
- ユーザーが少し前向きになれる一言を添える

3. 入力コンテキスト
- `SOUL.md`
- `USER.md`
- 前日会話履歴
- 当日予定一覧
- 実行日の日付情報

4. 前日会話履歴の扱い
- 少なくとも前日分の会話を読み、未完了話題、気分、悩み、直近の注力事項を拾う
- 朝挨拶の一言は、単なる定型励ましではなく、前日文脈に接続することを優先する
- 前日会話に参照価値が薄い場合は、不自然な言及を避ける

5. 当日予定の扱い
- 予定がある場合は、主要予定や時刻付き予定を分かりやすく要約する
- 予定が多い場合でも、朝メッセージ全体は冗長にしすぎない
- 予定がない場合は、その旨を自然に伝えたうえで、その日の過ごし方を押しつけずに支える文面にする

6. 出力要件
- ユーザーへそのまま送れる単一メッセージを返す
- 文面内に以下の要素を含める
  - 朝の挨拶
  - 今日の予定確認
  - 元気づける一言
- 予定列挙だけで終わらず、会話として自然な流れを持たせる
- 過度に長文化せず、朝に読みやすい長さに収める

## 文面方針
1. 挨拶
- 朝の時刻にふさわしい自然な第一声にする
- 既存の `SOUL.md` に反しない範囲で親しみや温度感を持たせる

2. 予定確認
- 「今日は何があるか」が一読で分かることを優先する
- 重要予定があれば先に触れる

3. 元気づける一言
- 画一的な応援ではなく、前日会話や `USER.md` の内容に沿わせる
- プレッシャーを増やす言い方や説教調は避ける
- 必要に応じて軽いねぎらい、期待、安心感のいずれかを含める

## 今回の成功条件
- 朝ジョブ文面を専用LLMが決める前提が明文化されている
- `SOUL.md` / `USER.md` / 前日会話履歴 / 当日予定を入力に含める要件が定義されている
- 「挨拶」「予定確認」「元気づける一言」の3要素が必須化されている
- 前日文脈に沿った自然な励ましを優先する方針が明記されている
- 当日予定がない場合の出力方針も定義されている

## 実装スコープ（Step 17）
1. 朝挨拶LLMの責務定義
- 朝ジョブで何を読み、何を返すかを定義する

2. 入力コンテキスト仕様の定義
- `SOUL.md`、`USER.md`、前日会話履歴、当日予定の利用方針を定義する

3. 出力文面要件の定義
- 朝の挨拶文に必要な構成要素と文体制約を定義する

4. 品質方針の定義
- 自然さ、簡潔さ、励まし方の制約を定義する

## 受け入れテスト（最小）
1. 朝ジョブ実行時に MorningGreetingComposer を呼ぶ想定になっている
2. 入力に `SOUL.md` と `USER.md` が含まれる想定になっている
3. 入力に前日会話履歴が含まれ、その文脈を朝メッセージへ反映できる想定になっている
4. 入力に当日予定一覧が含まれ、予定確認を文面に含める想定になっている
5. 出力が「挨拶」「予定確認」「元気づける一言」を含む想定になっている
6. 当日予定がゼロ件でも自然な文面を返せる想定になっている

## 非スコープ（Step 17時点）
- 朝ジョブ実装コードの変更
- MorningGreetingComposer の実際のプロンプト実装
- 朝メッセージ送信スケジュールの変更
- 励まし文面のABテストや評価基盤の構築

## 完了後の期待効果
- 朝の定時メッセージが固定通知ではなく、その日の文脈に沿った挨拶になる
- ユーザーが起床後にその日の予定を把握しやすくなる
- 前日からの流れを踏まえた一言で、朝の体験が少し人間的になる
